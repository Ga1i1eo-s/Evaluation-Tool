━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
POST /api/chat
  Body: { prompt: string, conversationHistory: Message[] }
  
  Step 1: Call Claude API to get the main response
  Stream the main response back to the frontend via SSE 
  (Server-Sent Events) so it types out in real time
  
  Step 2: After full response received, call Claude API again
  with this EVALUATION SYSTEM PROMPT:

--- EVALUATION SYSTEM PROMPT ---
You are an expert AI output evaluator. Analyze the AI response below 
and return ONLY a valid JSON object with this exact structure:

{
  "overallConfidence": "High" | "Moderate" | "Low",
  "overallSummary": "one sentence summary",
  "dimensions": {
    "correctness": { 
      "score": 1-5, 
      "label": "High"|"Medium"|"Low",
      "summary": "brief one-line summary",
      "detail": "2-3 sentence detailed explanation"
    },
    "completeness": { same structure },
    "reasoningQuality": { same structure },
    "usefulness": { same structure },
    "uncertainty": { same structure }
  },
  "keyFindings": {
    "assumptions": [
      { 
        "text": "quote from response that is an assumption",
        "explanation": "why this is an assumption and its risk",
        "spanText": "exact substring from response to underline"
      }
    ],
    "informationGaps": [
      { "gap": "description of missing info", "importance": "High"|"Medium"|"Low" }
    ],
    "risks": [
      { "risk": "description", "mitigation": "what user should do" }
    ],
    "strongPoints": [
      { "point": "description", "spanText": "exact substring" }
    ]
  },
  "suggestedPrompts": [
    "Follow-up prompt 1 to fill a gap",
    "Follow-up prompt 2 to verify an assumption", 
    "Follow-up prompt 3 to get more detail"
  ],
  "evaluationTrace": [
    { "step": "Checked factual claims", "result": "3 unverified statistics found", "dimension": "correctness" },
    { "step": "Assessed logical flow", "result": "Reasoning is generally sound", "dimension": "reasoningQuality" }
  ],
  "inlineAnnotations": [
    {
      "spanText": "exact substring from response to annotate",
      "type": "assumption" | "statistic" | "uncertain" | "strong",
      "tooltip": "tooltip text to show on hover"
    }
  ]
}

The original prompt was: {{PROMPT}}
The AI response to evaluate: {{RESPONSE}}
--- END SYSTEM PROMPT ---

  Return both the main response stream AND the evaluation JSON as 
  separate SSE events:
  - event: "chunk" → streaming response text
  - event: "evaluation" → complete evaluation JSON
  - event: "done" → signals completion

POST /api/confidence
  Body: { rating: number (1-5), responseId: string }
  Stores user's pre-clarity confidence rating

IMPORTANT IMPLEMENTATION NOTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. The main Claude response and the evaluation are TWO SEPARATE 
   API calls. Do NOT try to combine them into one call.

2. Stream the main response first using SSE. Only start the 
   evaluation call AFTER the full response text is received.

3. The annotation engine must use fuzzy substring matching 
   because the evaluation model may return slightly paraphrased 
   spanText. Use a simple contains() check first, fallback to 
   finding the closest match.

4. Never show raw JSON to the user. All evaluation data must 
   be rendered through the UI components.

5. The "Before Clarity" confidence dots must appear ABOVE the 
   response, immediately after it finishes streaming, BEFORE 
   the evaluation completes. The user rates first, then sees results.

6. Handle SSE errors gracefully — if evaluation fails, show 
   "Evaluation unavailable" state in the Clarity panel, not a crash.

7. Make the right panel collapsible with a toggle button on the 
   divider between the two panels (< > arrow).
